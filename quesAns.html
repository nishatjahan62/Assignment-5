Question-1: What are the different ways to select an element in the DOM?
Answer: There are some different ways to select an element in DOM.

i. getElementById : if we want to select only one element we use getElementById('the id name of the element')
[Ex: getElementById('card-1')]
ii. getElementsByClass : if we want to select some elements with same class name we use getElementsByClassName('the class name of the elements')
[Ex: getElementsByClass('btn-primary')]
iii. getElementsTagName : if we want to select elements by their tag name we use getElementsByTagName('the tag name of the element')
[Ex: getElementsByTagName('p')]
iv. querySelector : by querySelector we will find the first element which will match the selector. for this we use
if it is id querySelector(#id name)
if it is class querySelector(.class name)
[Ex: querySelector('#the id name')]
v. querySelectorAll : by querySelectorAll we will find all the elements which will match the selector. for this we use same as querySelector
if it is id querySelector(#id name)
if it is class querySelector(.class name)
[Ex: querySelector('.the class name')]
Question-2: What is the difference between innerHTML, innerText, and textContent ?
Answer: if we want to access the content of the html with DOM we use innerHTML , innerText and textContent. but there are few difference among them .

**innerHTML: BY innerHTML we can access the content of the element with the tag, texts etc. and the output will be just like vscode"s code.
**innerText: By innerText we can access only the visible text content of the elements and it will not includes any styles. and the output will only show the text.
**textContent: By Text content we can access the text content like innerText but it will also shows the hidden texts. and the output will be both visible and hidden texts.
Question-3: What is event delegation in the DOM?
Answer: Event delegation is a way to add event listener to a parent element instead of adding too many event listener to each child. In this way, we add one event listener to the parent and that event listener handle the child's event by event bubbling process.
Question-4: What is event bubbling in the DOM?
Answer: Event bubbling is a technique of event propagation where an event start from the target element to the parent. It first start from the element where the event is happening and move (bubbles up) to the parent then parent's parent and so on.
Question-5: How do you create, add, and remove elements using JavaScript?
Answer: In DOM we can create new elements, add elements also can remove elements. this is how we can do these

i. Create new elements For create a new element we use .createElement("name of the tag")
[Ex: const newElement=document.createElement('p')]
ii. Add new elements For add element we use .appendChild(the element)
[Ex: container.appendChild(newElement)]
iii. Remove elements to remove element we use .remove(). [Ex: newElement.remove()]
